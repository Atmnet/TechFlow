# TechFlow Visual Editor - 使用指南

## 🎨 功能概述

TechFlow Visual Editor 是一个基于拖拽的 Kubernetes 资源可视化编辑工具，支持：

- ✅ **拖拽式操作** - 直观的资源管理体验
- ✅ **动画效果** - 流畅的过渡和连接动画
- ✅ **实时 YAML 生成** - 拖拽同时生成 K8s 配置文件
- ✅ **资源连接** - 可视化定义服务间关系
- ✅ **多种资源类型** - Pod、Service、Deployment 等

---

## 🚀 快速开始

### 1. 访问编辑器

打开浏览器访问：http://localhost:3000/visual-editor

### 2. 添加资源

从左侧工具栏选择资源类型，点击即可添加到画布：

| 资源类型 | 图标 | 用途 |
|----------|------|------|
| Pod | 📦 | 基础容器单元 |
| Service | ⚙️ | 服务暴露 |
| Deployment | 🚀 | 部署管理 |
| ConfigMap | 📄 | 配置管理 |
| Secret | 🔐 | 敏感信息 |
| Database | 🗄️ | 数据库 |

### 3. 拖拽布局

- **移动资源**：拖拽资源卡片到目标位置
- **选择资源**：点击资源卡片
- **删除资源**：选中后点击"删除选中"按钮

### 4. 建立连接

1. 点击源资源的右侧连接点（白色圆点）
2. 点击目标资源的左侧连接点
3. 自动创建带动画的连接线

### 5. 查看/复制 YAML

右侧面板实时显示生成的 YAML 配置，点击"📋 复制"按钮可复制到剪贴板。

## 🎯 使用示例

### 示例 1：创建简单的 Web 应用

```
1. 添加 Deployment (nginx)
   - 配置：image: nginx:1.21, replicas: 3, port: 80

2. 添加 Service
   - 配置：port: 80, targetPort: 80

3. 建立连接
   - 从 Deployment 连接到 Service

4. 复制 YAML
   - 点击"📋 复制"获取配置
```

### 示例 2：创建带数据库的应用

```
1. 添加 Pod (app)
   - 配置：image: myapp:latest, port: 8080

2. 添加 Database
   - 配置：image: postgres:15, port: 5432

3. 添加 ConfigMap
   - 存储数据库连接配置

4. 建立连接
   - Pod → Database (应用连接数据库)
   - ConfigMap → Pod (配置注入)

5. 生成 YAML
   - 包含所有资源的完整配置
```

## 🔧 技术细节

### 动画效果

- **拖拽动画**：使用 Framer Motion 的 `drag` 属性
- **缩放效果**：`whileDrag` 和 `whileHover` 状态
- **连接动画**：SVG 路径动画 + 虚线流动效果

### 连接线

- **贝塞尔曲线**：平滑的 S 形连接
- **渐变颜色**：蓝紫色渐变
- **流动动画**：循环路径动画表示数据流
- **箭头指示**：方向性标识

### YAML 生成

```typescript
// 根据节点类型生成对应的 K8s 资源
{
  apiVersion: "v1",
  kind: "Pod",
  metadata: { name: "pod-1" },
  spec: {
    containers: [{
      name: "pod-1",
      image: "nginx:latest",
      ports: [{ containerPort: 80 }]
    }]
  }
}
```

| 元素 | 颜色 | 用途 |
|------|------|------|
| Pod | 蓝色 (#3B82F6) | 基础容器 |

### 交互反馈

- **悬停**：放大 5% + 阴影增强
- **拖拽**：放大 10% + 光标变化
- **选中**：白色发光边框
- **连接**：蓝色提示框

## 🐛 已知限制

1. **YAML 格式**：当前使用 JSON.stringify 简化实现，建议使用 js-yaml 库
2. **连接验证**：未验证连接的合法性（如 Pod 不能直接连接 Secret）
3. **配置编辑**：暂不支持在界面上编辑资源详细配置
4. **持久化**：刷新页面后数据丢失
