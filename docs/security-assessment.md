## 1. 执行摘要

本报告评估使用 WebContainers API 构建 K8s YAML 验证沙盒的安全性。WebContainers 是在浏览器中运行 Node.js 的技术，由 StackBlitz 开发。

### 评估结论

| 维度 | 评级 | 说明 |
|------|------|------|
| 隔离性 | ✅ 良好 | 浏览器沙盒天然隔离 |
| 代码执行 | ⚠️ 中等 | 需限制命令执行范围 |
| 数据安全 | ✅ 良好 | 无持久化存储风险 |
| 网络访问 | ✅ 良好 | 浏览器同源策略保护 |
| 资源消耗 | ⚠️ 中等 | 需限制 CPU/内存使用 |

**整体风险等级**: 🟡 中等（可接受）

## 2. 架构安全分析

```

           ↓ 受控 API 调用
```

## 3. 威胁建模

### 3.1 威胁场景

#### T1: 恶意 YAML 注入
- **描述**: 用户上传恶意构造的 YAML 尝试执行任意代码
- **可能性**: 中
- **影响**: 高
- **缓解措施**:
  - YAML 解析前进行语法验证
  - 限制可执行的 kubectl 命令白名单
  - 沙盒内禁用 node:child_process

#### T2: 资源耗尽攻击
- **描述**: 大量请求导致沙盒资源耗尽
- **可能性**: 中
- **影响**: 中
- **缓解措施**:
  - 实现请求速率限制
  - 沙盒启动超时控制
  - 最大并发沙盒数限制

#### T3: 沙盒逃逸
- **描述**: 尝试突破 WebContainer 沙盒限制
- **可能性**: 低
- **影响**: 高
- **缓解措施**:
  - 使用最新版 WebContainers
  - 定期安全更新
  - 监控异常行为

#### T4: 数据泄露
- **描述**: 敏感 YAML 内容被未授权访问
- **可能性**: 低
- **影响**: 中
- **缓解措施**:
  - 传输层加密 (HTTPS)
  - 验证记录加密存储
  - 访问日志审计

### 3.2 威胁矩阵

| 威胁 | 可能性 | 影响 | 风险值 | 优先级 |
|------|--------|------|--------|--------|
| T1: 恶意注入 | 中 | 高 | 高 | P0 |
| T2: 资源耗尽 | 中 | 中 | 中 | P1 |
| T3: 沙盒逃逸 | 低 | 高 | 中 | P1 |
| T4: 数据泄露 | 低 | 中 | 低 | P2 |

## 4. 安全控制措施

### 4.1 输入验证

```javascript
// YAML 内容验证
const yamlSchema = z.string()
  .max(1024 * 1024, 'YAML 内容不能超过 1MB')
  .refine(content => {
    // 禁止包含危险关键字
    const dangerous = ['eval(', 'exec(', 'require("child_process")'];
    return !dangerous.some(k => content.includes(k));
  }, 'YAML 包含危险内容');
```

### 4.2 命令白名单

```javascript
// 允许的 kubectl 命令
const ALLOWED_COMMANDS = [
  'kubectl version',
  'kubectl explain',
  'kubectl validate',
  'kubectl apply --dry-run=client',
];

function isCommandAllowed(cmd, args) {
  const fullCmd = `${cmd} ${args?.join(' ') || ''}`;
  return ALLOWED_COMMANDS.some(allowed => 
    fullCmd.startsWith(allowed)
  );
}
```

### 4.3 速率限制

```javascript
// 使用 @fastify/rate-limit
await fastify.register(require('@fastify/rate-limit'), {
  max: 100,
  timeWindow: '1 minute',
  allowList: ['127.0.0.1'],
});
```

### 4.4 资源限制

| 资源 | 限制 | 实现方式 |
|------|------|----------|
| 沙盒启动时间 | < 2 秒 | 超时控制 |
| 命令执行时间 | < 30 秒 | spawnSync timeout |
| 并发沙盒数 | ≤ 10 | 信号量控制 |
| YAML 大小 | ≤ 1MB | 输入验证 |
| 请求频率 | ≤ 100/min | 速率限制 |

## 8. 结论
